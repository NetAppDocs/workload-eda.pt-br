= Guia rápido do Workload Factory para EDA
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
Comece a usar o Workload Factory para EDA configurando seu ambiente, ativando os painéis de monitoramento e configurando a análise de latência para otimizar o desempenho do FSx para ONTAP em suas cargas de trabalho de EDA.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["Um"] Faça login no Workload Factory
[role="quick-margin-para"]
Você precisará de https://docs.netapp.com/us-en/workload-setup-admin/sign-up-saas.html["Crie uma conta no Workload Factory."^] e faça login usando um dos https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["experiências de console"^].

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["Dois"]Adicionar credenciais e permissões
[role="quick-margin-para"]
Escolha as políticas de permissão que melhor atendam às suas necessidades.

[role="quick-margin-para"]
Se optar por não conceder permissões, você poderá começar a usar o Workload Factory for EDA para copiar exemplos de código parcialmente concluídos.

[role="quick-margin-para"]
Se optar por conceder permissões, terá de adicionar manualmente as credenciais a uma conta, o que inclui selecionar as capacidades de carga de trabalho, como EDA e IA, e criar as políticas IAM para as permissões necessárias.

[role="quick-margin-para"]
https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["Aprenda como adicionar credenciais e permissões."^].

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["Três"] Configure seu ambiente para atender aos requisitos da EDA.
[role="quick-margin-para"]
Você precisará de um sistema de arquivos FSx para ONTAP implantado e detectado que contenha pelo menos um volume configurado como um compartilhamento NFS.

[role="quick-margin-para"]
link:requirements-eda.html["Saiba mais sobre os requisitos de EDA"^].

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["Quatro"]Ative o painel de visão geral
[role="quick-margin-para"]
Ative o painel Visão geral para monitorar a integridade, a capacidade e a taxa de transferência dos seus clusters FSx for ONTAP. Aceite a solicitação de consentimento para permitir que o Workload Factory comece a coletar métricas do CloudWatch.

[role="quick-margin-para"]
link:task-activate-overview-dashboard.html["Aprenda como ativar o painel de Overview"^].

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png["Cinco"]Configurar o painel de Projetos
[role="quick-margin-para"]
Configure o painel de controle de Projetos para monitorar a integridade do sistema, o desempenho e o uso de storage, permitindo que você otimize seu ambiente EDA de forma eficaz.

[role="quick-margin-para"]
link:task-configure-project-dashboard.html["Aprenda a configurar o painel de controle de Projetos"^].

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-6.png["Seis"]Configurar FSx para análise de latência do ONTAP
[role="quick-margin-para"]
Defina limites de latência para identificar e resolver proativamente problemas de desempenho antes que eles afetem o tempo de execução da simulação. Configure alertas de aviso e críticos, configure notificações por e-mail ou SNS para receber alertas em tempo real, aproveite a análise básica automatizada e, opcionalmente, habilite a análise de IA para cenários complexos configurando um ARN de modelo do Amazon Bedrock.

[role="quick-margin-para"]
link:task-configure-latency-monitoring.html["Aprenda como configurar o monitoramento de latência"^]. Após a detecção de eventos, você pode link:task-analyze-latency-trends.html["analisar tendências de latência"^] usar o gráfico interativo.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-7.png["Sete"] Integrar com outros serviços
[role="quick-margin-para"]
Integre o EDA com outros serviços, como um portal de serviços como serviço ou o Perforce Helix Core, usando a API REST.

[role="quick-margin-para"]
link:integrate-perforce.html["Aprenda como integrar EDA com Perforce."^].
